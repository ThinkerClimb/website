解決"crt1.o: No such file or directory"問題

編譯嵌入式系統程式的使用者一定有遇過"crt1.o: No such file or directory"的問題，編譯器提示找不到相關的crt檔案，可是該toolchain明明就已經存在這個檔案，為何toolchain還是提示找不到呢？其實原因在於export bin路徑的lib資料夾(與bin資料夾平行的lib資料夾)並沒有crt相關檔案，因此使用只需要將該toolchain的crt相關檔案複製到與bin資料夾平行的lib資料夾裡面即可，當然使用者也可以使用連結路徑的方式。

或者在LDFLAGS指定sysroot路徑即可： 

LDFLAGS = --sysroot=/opt/mipsel-linux-uclibc/

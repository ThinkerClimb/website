SNK NeoGeo X v370 >> Dingoo

安裝Dingoo(SDCard0)

由於司徒的NeoGeo X掌機是v370版本，所以內部不是使用MicroSD，而是已經改成Flash Memory IC了，而這樣的更改，讓司徒無法去修改內部系統檔案，因此要把Dingoo系統安裝上去，真的是有一點困難，雖然司徒已經改機成可以使用SDCard-0開機，但是，司徒沒有SDCard-0的燒錄檔案，而NeoGeo X Forum雖然有提供root file system、sys_update_file(uboot、kernel、app file system、config file system)等下載檔案，但是，最重要的root file system卻是使用7-zip打包，因為使用7-zip打包會把Linux系統的捷徑改掉，這樣會導致系統還原有問題，所幸有一位大陸網友(Lawrence)提供MicroSD的映像檔案給司徒，才終於讓司徒成功替換成Dingoo系統。

司徒就將替換過程說明如下(需要Windows系統和Ubuntu系統)：
1. 下載燒錄檔案(MicroSD、SDCard-0適用)
2. 準備一張2GB MicroSD
3. 使用WinHEX Clone燒錄檔案到2GB MicroSD
4. 下載RZX-50_fw_121126.rar
5. 將RZX-50_fw_121126.rar解壓縮並且複製到Ubuntu系統的根目錄

# cd /tmp
# mkdir appfs
# mkdir rootfs
# mkdir configfs
# mount -o loop /RZX-50_fw_121126/appfs.ext3 /tmp/appfs
# mount -o loop /RZX-50_fw_121126/rootfs.ext3 /tmp/rootfs
# mount -o loop /RZX-50_fw_121126/configfs.ext3 /tmp/configfs

6. 將2GB MicroSD插入到Ubuntu系統中，應該會出現如下畫面

7. 使用gparted命令將20M File System格式化成ext3檔案系統
8. 使用gparted命令將130M File System格式化成ext3檔案系統

9. 複製/tmp/appfs全部內容並覆蓋到130M File System(/dev/sdf2)
10. 複製/tmp/configfs全部內容複製並覆蓋到20M File System(/dev/sdf3)
11. 複製/tmp/rootfs全部內容複製並覆蓋到480M File System(/dev/sdf1)
12. 全部資料複製完成後，將MicroSD插入到Neo Geo X掌機的SDCard-0並且重新開機

在Ubuntu系統複製檔案時， 如果出現Error請使用Skip All

成功開機後，Dingoo系統會運行於俄國語言，為了擁有更完美的Dingoo系統，司徒已經將Dingoo系統漢化，改成繁體中文。

最後，司徒修改了Dingoo內部相關設定，讓Dingoo系統可以支援英文、繁體中文兩種語言，而模擬器目前也支援：
PS1模擬器
FC模擬器
SFC模擬器
SEGA MD模擬器
SEGA SMS模擬器
GBA模擬器
